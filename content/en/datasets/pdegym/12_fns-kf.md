# FNS-KF: Forced Incompressible Kolmogorov Flow

**Description:** Incompressible Navier–Stokes trajectories driven by a fixed sinusoidal Kolmogorov forcing. This task adds external forcing as a static conditioning field. Initial velocity follows NS-PwC, while persistent forcing changes the subsequent dynamics.

**Dataset authors/maintainers:** The POSEIDON authors, Computational and Applied Mathematics Laboratory, ETH Zurich.
**Code or software used to generate the data:** AZEBAN for the flow; forcing generated by the official loader.

## Equation

$$
\partial_t\mathbf u+(\mathbf u\cdot\nabla)\mathbf u+\nabla p-\nu\Delta\mathbf u=\mathbf f,
\qquad \nabla\cdot\mathbf u=0,
$$
with the static forcing used in the paper
$$
f(x,y)=0.1\sin(2\pi(x+y)).
$$
The official operator implementation includes this field as an extra channel with $f_t=0$.

### Physical quantities

- Coordinates are two-dimensional Cartesian $(x,y)$. Unless stated otherwise, coordinates are implicit in array indices and are not extra channels.
- Raw release fields, paper operator fields, and official model-facing channels can differ; see “Data format and channels.”

## Operator-learning task

$[u_x,u_y,f](0)\mapsto[u_x,u_y,f](t)$; evaluation is on velocity only.

## Adjustable, varied, and fixed parameters

| Parameter | Meaning / type | Status in the release | Value or distribution |
|---|---|---|---|
| $c_{ij}$ | initial piecewise-constant vorticity; IC / data-distribution parameter | varied per trajectory | $\mathcal U[-1,1]$ |
| forcing amplitude | forcing amplitude; PDE/source parameter | fixed | $0.1$ |
| forcing wavevector | forcing wave number and direction; PDE/source parameter | fixed | $2\pi(x+y)$ |
| time dependence | forcing time dependence; PDE/source parameter | fixed as static | $f_t=0$ |
| $\nu$ | viscosity/spectral stabilization; PDE/numerical coefficient | fixed | $\simeq4\times10^{-4}$ |

**Summary:** Forcing amplitude, wave number, direction, phase, time dependence, and viscosity are adjustable; the release fixes the forcing and varies only velocity initial data.

> “Adjustable” has three distinct meanings: mathematically changeable in the PDE, changeable by editing the generator, and actually sampled in the official release. Random initial-condition parameters are often encoded only in the field realization rather than stored as an explicit metadata vector.

## Data format, input/output sizes, and channels

### Raw release

- Raw shape: `solution: [20000, 21, 2, 128, 128]`
- Raw channels/variables: Stores only $[u_x,u_y]$; forcing is reconstructed analytically by the loader.
- Expected assembled filename: `FNS-KF.nc`

### Official POSEIDON model interface

- One input/output pair: `[5,128,128] → [5,128,128]`
- Channel definition: $[1,u_x,u_y,0,f]_{t_i}\to[1,u_x,u_y,0,f]_{t_j}$.
- After batching the model generally sees `[B,C,H,W]`; trajectory files are typically `[N,T,C,H,W]`. These should not be conflated.
- For time-dependent tasks, `BaseTimeDataset` returns two snapshots and a scalar lead time. all2all pairing enumerates all selected pairs with $t_i\le t_j$.

## Numerical generation

AZEBAN for the flow; forcing generated by the official loader.

Normalization is only a training implementation detail: the official loader applies precomputed means and standard deviations, which neither adds physical fields nor constitutes a PDE-parameter variation.

## Download and assembly

```bash
pip install -U huggingface_hub
huggingface-cli download camlab-ethz/FNS-KF --repo-type dataset --local-dir ./FNS-KF
```

Enter the downloaded directory and run the included assembly script:

```bash
cd ./FNS-KF
python assemble_data.py --input_dir . --output_file FNS-KF.nc
```

The assembled path can be passed to the official training/inference scripts through `--data_path`. A steady identifier may be suffixed with `.time` to wrap the steady map as a normalized lead-time-one long-time-limit task; this does not create a physical trajectory in the raw file.

## What is interesting and challenging

- The model must distinguish a static PDE/source condition from dynamic velocity while learning persistent energy injection.
